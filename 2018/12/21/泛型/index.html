一、使用初衷（场景）

1、我们都希望我们的代码复用性高，而在 Java 中，我们想要编写可以应用于多种类型的代码，就需要用到泛型了。

2、仅一次方法调用就能返回多个对象。

3、而且在研读一些成熟的框架源码的时候，会发现源码存在着大量的泛型使用，不了解好泛型，很难理解得了框架设计。

二、泛型的简单使用

泛型是在 Java5 提出的，泛型实现了参数化类型（类型参数化）的概念。在你创建参数化类型的一个实例时，编译器会为你负责转型操作，并且保证类型的正确性。

有些时候，我们会希望容器可以同时支持多种类型对象，但是，通常而言，我们只会使用容器来存储一种类型对象。泛型存在目的之一便是解决这个给容器类指定存放特定类型。

看一下如何通过泛型来实现避免了重复创建对象的问题。

可以看到，我们如果想要存入什么类型的对象，只需要在创建泛型类的时候指定对象的数据类型即可。

当创建泛型对象的时候，必须在尖括号内指明想持有什么类型的对象，就只能在泛型容器类中存入该类型，在编译期间，编译器会自动帮我们校验存入的数据类型是否正确。

为什么不适用Object 作为数据类型，而选用泛型呢？

  1. 在 Java SE5 之前，将一个对象放到容器类的时候，这个对象会被向上转型为 Object，因此会丢失类型信息（需要开发者自己记得存入的是什么类型）。
  2. 当想要将这个对象从容器中取回，用它去执行某些操作的时候，必须将其向下转型回正确的类型。
  3. 放到容器里的和从容器中取回的，都是 Object。因此，我们很可能会将一个不同类型的对象放入到另外一个类型的对象容器中去。

那总结一下泛型的特点呗：

  • 指定容器类要持有什么类型的对象。
  • 由编译器来保证类型的正确性。

使用泛型类和泛型方法有个区别

使用泛型类时候，必须在创建对象的时候指定类型参数的值，而使用泛型方法的时候，通常不必指明参数类型，因为编译器会为我们找出具体的类型。这称为 类型参数推断 。

so，在泛型中，有一个残酷的现实是：

在泛型代码内部，无法获取任何有关泛型参数类型的信息

Java 泛型是使用擦除来实现的，这意味着当你再使用泛型的时候，任何具体类型信息都被擦除，你唯一知道的就是你在使用一个对象。因此 List<String>和 List<Integer> 在运行的时事实上都是相同类型。这两种形式都被擦除为它原生的类型，即 List。
