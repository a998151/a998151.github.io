Java 中的 ==、equals 与 hashCode

==

  • 比较两个基本类型的数据是否相等
  • 两个引用变量是否相等

​ 如果两个变量是基本数据类型，可以直接使用“==”运算符来比较其对应的值是否相等。

​ 但是不可以用 ”==“ 来比较应用类型变量的值是否相等。具体原因如下：

​ 如果一个变量指向的数据是对象，那么这时候就涉及了两块内存，对象本身占用了一块内存（堆内存），变量也占用了一块内存，例如

​ 变量 str 占用一块内存，而 new String() 占用另外一块内存，此时变量 str 所对应内存中存储的数值就是对象占用的那块内存的首地址。

​ 对于指向对象类型的变量，如果想要比较两个变量是否指向同一对象，即要看这两个变量所对应的内存中的数值是否相等（这两个对象是否指向同一块存储空间），这时候就可以用“==”运算符进行比较。但是，如果要比较这两个对象的内容是否相等，那么用“==”运算符就无法实现了。

equals

​ equals()是 Object 类提供的方法之一。每一个 Java 类都集成 Object 类，所以每一个对象都有 equals 这个方法。

​ Object类中定义的equals(Object)方法的情况下，equals(Object)与“==”运算符一样，比较的是引用。

​ 相比“==”运算符，equals(Object)方法的特殊之处就在于它可以被覆盖，所以可以通过覆盖的方法让它不是比较引用而是比较数据内容，例如String类的equals方法是用于比较两个独立对象的内容是否相同，即堆中的内容是否相同

​ 如果一个类没有自己定义equals()方法，它默认的equals()方法（从Object类继承的）就是使用“==”运算符，也是在比较两个变量指向的对象是否是同一对象，此时使用equals()方法和使用“==”运算符会得到同样的结果。

hashCode

所有散列函数都有如下一个基本特性：如果两个散列值是不相同的（根据同一函数），那么这两个散列值的原始输入也是不相同的。

而散列表就是一个哈希函数的主要使用。在 Java 中，散列表的实现是 HashMap，具体可以看对应文章，都介绍的非常详细了。在实现的过程中，判断一个对象是否相等，会先判断其 hashCode 是否相等再使用 equal 方法进行判断。

这里，我们就主要介绍

  • HashCode 是什么
  • HashCode 有什么作用
  • 不同对象的 HashCode 一定不同吗？

Java中的hashCode方法就是根据一定的规则将与对象相关的信息（比如对象的存储地址，对象的字段等）映射成一个数值，这个数值称作为散列值。

有些朋友误以为默认情况下，hashCode返回的就是对象的存储地址，事实上这种看法是不全面的，确实有些JVM在实现时是直接返回对象的存储地址，但是大多时候并不是这样，只能说可能存储地址有一定关联。下面是HotSpot JVM中生成hash散列值的实现：

因此有人会说，可以直接根据hashcode值判断两个对象是否相等吗？

肯定是不可以的，因为不同的对象可能会生成相同的hashcode值。虽然不能根据hashcode值判断两个对象是否相等，但是可以直接根据hashcode值判断两个对象不等，如果两个对象的hashcode值不等，则必定是两个不同的对象。如果要判断两个对象是否真正相等，必须通过equals方法。

也就是说对于两个对象，如果调用equals方法得到的结果为true，则两个对象的hashcode值必定相等；

如果equals方法得到的结果为false，则两个对象的hashcode值不一定不同；

如果两个对象的hashcode值不等，则equals方法得到的结果必定为false；

如果两个对象的hashcode值相等，则equals方法得到的结果未知。

在重写hashCode方法的时候，要注意以下的点：

  • 在有些情况下，程序设计者在设计一个类的时候为需要重写equals方法，比如String类，但是千万要注意，在重写equals方法的同时，必须重写hashCode方法。

　“设计hashCode()时最重要的因素就是：无论何时，对同一个对象调用hashCode()都应该产生同样的值。如果在讲一个对象用put()添加进HashMap时产生一个hashCdoe值，而用get()取出时却产生了另一个hashCode值，那么就无法获取该对象了。

​ 所以如果你的hashCode方法依赖于对象中易变的数据，用户就要当心了，因为此数据发生变化时，hashCode()方法就会生成一个不同的散列码”。

——《Java编程思想》

小结

  • Java中的hashCode方法就是根据一定的规则将与对象相关的信息（比如对象的存储地址，对象的字段等）映射成一个数值，这个数值称作为散列值。
  • hashCode 返回的只能说可能存储地址有一定关联，但是大多 JVM 的实现都不是返回对象的存储地址。
  • 不可以根据 hashCode 判断两个对象是否相等，但可以用来判断两个对象是否为不同的两个对象。
